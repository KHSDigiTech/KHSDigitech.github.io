1. Why Programs Crash - The Problem We're Solving

In previous weeks, we've written programs that expect users to enter exactly what we ask for. But what happens when they don't? Let's see what goes wrong:

‚ùå A Program That Will Crash:

# This program expects a number but will crash if user enters text
age = int(input("Enter your age: "))  # User types "hello"
print(f"You are {age} years old")

# Result: ValueError: invalid literal for int() with base 10: 'hello'
# Program terminates immediately!

The Problem: When the user enters "hello", int() can't convert it to a number, so Python throws a ValueError and the program crashes. This is unprofessional and creates a poor user experience.

Real-World Impact:

Imagine if your calculator app crashed every time someone accidentally pressed a letter instead of a number. Or if an online shopping cart crashed when someone entered their phone number incorrectly. Error handling is what separates amateur programs from professional software.

2. The try/except Solution - Catching Errors Gracefully

The try/except block is Python's way of saying "try this code, but if it fails, don't crash - do this instead." It's like having a safety net for your program.

How try/except Works:

  1. try: Python attempts to run the code in this block
  2. If no error occurs: The except block is skipped entirely
  3. If an error occurs: Python immediately jumps to the except block
  4. Program continues: After the except block, the program continues normally

‚úÖ The Same Program, Made Robust:

# Now the program handles errors gracefully
try:
    age = int(input("Enter your age: "))
    print(f"You are {age} years old")
except ValueError:
    print("Sorry, that's not a valid number. Please try again.")

# Now if user types "hello", we get a friendly message instead of a crash!

3. Understanding Different Error Types

Different operations can cause different types of errors. Understanding these helps you write more specific error handling:

# ValueError - Invalid conversion
try:
    number = int("hello")  # Can't convert "hello" to integer
except ValueError:
    print("That's not a number!")

# ZeroDivisionError - Division by zero
try:
    result = 10 / 0  # Can't divide by zero
except ZeroDivisionError:
    print("Cannot divide by zero!")

# IndexError - List index out of range
try:
    my_list = ["a", "b", "c"]
    print(my_list[10])  # Index 10 doesn't exist
except IndexError:
    print("That index doesn't exist in the list!")

4. Multiple except Blocks - Handling Different Errors Differently

You can have multiple except blocks to handle different types of errors with specific responses:

# A calculator that handles multiple error types
try:
    first_number = int(input("Enter first number: "))
    second_number = int(input("Enter second number: "))
    result = first_number / second_number
    print(f"Result: {result}")

except ValueError:
    print("Error: Please enter valid numbers only.")
    
except ZeroDivisionError:
    print("Error: Cannot divide by zero. Please enter a non-zero second number.")

5. Input Validation Loops - Keep Asking Until Valid

Professional programs don't just give an error message and quit. They ask the user to try again. We can combine loops with try/except to create robust input validation:

# Keep asking until user enters a valid number
while True:
    try:
        age = int(input("Enter your age: "))
        break  # Exit the loop if conversion succeeds
    except ValueError:
        print("That's not a valid number. Please try again.")
        # Loop continues, asking again

print(f"Thank you! You are {age} years old.")

How the Validation Loop Works:

  1. while True: creates an infinite loop
  2. Try to get and convert user input
  3. If successful, break exits the loop
  4. If error occurs, print message and loop continues
  5. User gets another chance to enter valid input

6. Creating Reusable Input Functions

We can create functions that handle input validation for us, making our code cleaner and more reusable:

def get_integer_input(prompt):
    """Keep asking for input until user enters a valid integer"""
    while True:
        try:
            value = int(input(prompt))
            return value  # Return the valid integer
        except ValueError:
            print("Please enter a valid whole number.")

def get_positive_integer(prompt):
    """Get a positive integer (greater than 0)"""
    while True:
        try:
            value = int(input(prompt))
            if value <= 0:
                print("Please enter a positive number (greater than 0).")
                continue
            return value
        except ValueError:
            print("Please enter a valid whole number.")

# Using our robust input functions
age = get_positive_integer("Enter your age: ")
num_siblings = get_integer_input("How many siblings do you have? ")

Example 3: Menu System with Error Handling

# Professional menu system with error handling
def show_menu():
    print("\n=== Student Grade Calculator ===")
    print("1. Add grade")
    print("2. View average")
    print("3. Exit")

def get_menu_choice():
    while True:
        try:

            else:

grades = []

while True:
    show_menu()
    choice = get_menu_choice()

        try:
            grade = float(input("Enter grade (0-100): "))

                print(f"Grade {grade} added successfully!")

                print("Grade must be between 0 and 100.")
        except ValueError:
            print("Please enter a valid number.")

            print(f"Your average grade is: {average:.2f}")
        else:

üö® Common Error Handling Patterns

‚ùå Don't Do This - Hiding All Errors:

try:
    # Some code here
    pass
except:  # Catches ALL errors - bad practice!
    pass   # Silently ignores errors - very bad!

Why this is bad: This hides all errors, making debugging impossible. You won't know what went wrong or why.

‚úÖ Do This - Specific Error Handling:

try:
    age = int(input("Age: "))
except ValueError as e:  # Catch specific error type
    print(f"Invalid input: {e}")  # Provide helpful message

Why this is good: Catches only the specific error we expect and provides meaningful feedback.

Best Practices for Error Handling:

  • Be specific: Catch specific error types, not all errors
  • Provide feedback: Tell the user what went wrong and how to fix it
  • Keep trying: Use loops to give users multiple attempts
  • Validate input: Check if input makes sense (e.g., age > 0)
  • Plan for errors: Think about what could go wrong and handle it
