1. The Problem with Simple Lists

Lists are great for storing multiple items, but they have limitations when storing complex information. Consider storing contact information:

‚ùå Problems with List-Only Approach:

# Using only lists - confusing and error-prone
contact = ["Alice Smith", "027-123-4567", "alice@email.com"]

# What does contact[1] mean? Phone or email?
print(contact[1])  # Is this phone or email? Hard to remember!

# What if we want to add more information?
# Order becomes critical and confusing

Issues: Hard to remember what index means what, difficult to add new fields, not self-documenting code.

2. What is a Dictionary?

A dictionary is a collection of key-value pairs. Instead of accessing data by position (like lists), you access data by meaningful names (keys). It's like a real dictionary where you look up a word (key) to find its definition (value).

Real-World Analogy:

A dictionary is like a contact card or a filing system. Instead of remembering "Alice's phone number is in slot 2," you simply ask for "Alice's phone number" directly.

Dictionary Structure Visualization:

contact = {
    "name": "Alice Smith",      ‚Üê Key: "name",    Value: "Alice Smith"
    "phone": "027-123-4567",    ‚Üê Key: "phone",   Value: "027-123-4567"
    "email": "alice@email.com"  ‚Üê Key: "email",   Value: "alice@email.com"
}

Access: contact["name"] ‚Üí "Alice Smith"
        contact["phone"] ‚Üí "027-123-4567"

3. Creating Dictionaries

Dictionaries are created using curly braces {} with key-value pairs separated by colons.

# Creating an empty dictionary
empty_dict = {}
student_grades = {}

# Creating a dictionary with initial data
contact = {
    "name": "Alice Smith",
    "phone": "027-123-4567",
    "email": "alice@email.com",
    "age": 25
}

# Dictionaries can store different data types
student = {
    "name": "Bob Johnson",
    "age": 16,
    "grades": [85, 92, 78],
    "is_enrolled": True
}

print(contact)  # Shows the entire dictionary

Dictionary Creation Rules:

  1. Use curly braces {} to define a dictionary
  2. Separate key and value with a colon :
  3. Separate pairs with commas
  4. Keys must be strings (in quotes) or numbers
  5. Values can be any data type

4. Accessing Dictionary Values

Use square brackets with the key name to access values. This is much clearer than remembering list positions!

# Accessing dictionary values
contact = {
    "name": "Alice Smith",
    "phone": "027-123-4567",
    "email": "alice@email.com"
}

# Clear, self-documenting access
name = contact["name"]
phone = contact["phone"]
email = contact["email"]

print(f"Name: {name}")
print(f"Phone: {phone}")
print(f"Email: {email}")

# Using in user interaction
print(f"Calling {contact['name']} at {contact['phone']}...")

‚úÖ Dictionary Benefits Over Lists:

  • Self-documenting: Code clearly shows what data you're accessing
  • Flexible: Easy to add new fields without breaking existing code
  • Robust: No need to remember numeric positions
  • Professional: Industry standard for structured data

6. Checking if Keys Exist

Before accessing a key, it's good practice to check if it exists to avoid errors.

# Safe key checking
contact = {"name": "Alice", "phone": "123-456"}

# Method 1: Using 'in' operator
if "email" in contact:
    print(f"Email: {contact['email']}")
else:
    print("No email address on file")

# Method 2: Using .get() method with default value
email = contact.get("email", "No email provided")
print(f"Email: {email}")

# Interactive field checking
field = input("What information do you want? (name/phone/email): ")
if field in contact:
    print(f"{field.title()}: {contact[field]}")
else:
    print(f"Sorry, no {field} information available.")

7. Deleting Dictionary Entries

Use the del keyword to remove key-value pairs from a dictionary.

# Removing dictionary entries
contact = {
    "name": "Alice Smith",
    "phone": "027-123-4567",
    "email": "alice@email.com",
    "temp_note": "Delete this later"
}

print("Before deletion:", contact)

# Safe deletion - check if key exists first
if "temp_note" in contact:
    del contact["temp_note"]
    print("Temporary note removed.")

print("After deletion:", contact)

# Interactive deletion
field_to_remove = input("Which field to remove? ")
if field_to_remove in contact:
    del contact[field_to_remove]
    print(f"Removed {field_to_remove} from contact.")
else:
    print(f"{field_to_remove} not found in contact.")

8. Looping Through Dictionaries

Dictionaries provide several methods for iteration: .items(), .keys(), and .values().

# Dictionary for looping examples
contact = {
    "name": "Alice Smith",
    "phone": "027-123-4567",
    "email": "alice@email.com",
    "age": 25
}

# Method 1: Loop through key-value pairs
print("=== Contact Information ===")
for key, value in contact.items():
    print(f"{key.title()}: {value}")

# Method 2: Loop through keys only
print("\n=== Available Fields ===")
for key in contact.keys():
    print(f"- {key}")

# Method 3: Loop through values only
print("\n=== All Values ===")
for value in contact.values():
    print(f"- {value}")

# Practical example: Display formatted contact card
print("\n=== Contact Card ===")
for field, info in contact.items():
    print(f"{field.upper():<10}: {info}")
Method Returns Use When Example
.items() Key-value pairs Need both key and value Displaying formatted data
.keys() Keys only Need field names Showing available options
.values() Values only Need just the data Calculating totals/averages

10. Practical Dictionary Operations

Let's explore common operations you'll need for your assessment program.

# Adding new contacts to the list
def add_contact(contacts_list):
    new_contact = {}
    new_contact["name"] = input("Enter name: ")
    new_contact["phone"] = input("Enter phone: ")
    new_contact["email"] = input("Enter email: ")
    
    contacts_list.append(new_contact)
    print(f"Added {new_contact['name']} to contacts.")

# Finding a contact by name
def find_contact(contacts_list, search_name):
    for contact in contacts_list:
        if contact["name"].lower() == search_name.lower():
            return contact
    return None

# Count total contacts
def count_contacts(contacts_list):
    return len(contacts_list)

# Example usage
contacts = []
add_contact(contacts)
add_contact(contacts)

print(f"\nTotal contacts: {count_contacts(contacts)}")

search_name = input("Search for contact: ")
found_contact = find_contact(contacts, search_name)

if found_contact:
    print("Found contact:")
    for key, value in found_contact.items():
        print(f"  {key.title()}: {value}")
else:
    print(f"No contact found with name '{search_name}'")

Example 1: Contact Book (Assessment-Style)

# Complete contact book using dictionaries - Merit level structure
contacts = []

def display_menu():
    print("\n=== Contact Book ===")
    print("1. Add contact")
    print("2. View all contacts")
    print("3. Search contacts")
    print("4. Exit")

def add_contact():
    print("\n--- Add New Contact ---")
    new_contact = {
        "name": input("Name: "),
        "phone": input("Phone: "),
        "email": input("Email: ")
    }
    
    contacts.append(new_contact)
    print(f"‚úì Added {new_contact['name']} to contacts.")

def view_all_contacts():
    if len(contacts) == 0:
        print("\nNo contacts found.")
        return
    
    print(f"\n=== All Contacts ({len(contacts)} total) ===")
    for i, contact in enumerate(contacts, 1):
        print(f"\nContact {i}:")
        for field, value in contact.items():
            print(f"  {field.title()}: {value}")

def search_contacts():
    if len(contacts) == 0:
        print("\nNo contacts to search.")
        return
    
    search_term = input("\nEnter name to search for: ").lower()
    found_contacts = []
    
    for contact in contacts:
        if search_term in contact["name"].lower():
            found_contacts.append(contact)
    
    if found_contacts:
        print(f"\nFound {len(found_contacts)} matching contact(s):")
        for contact in found_contacts:
            print(f"\n{contact['name']}:")
            for field, value in contact.items():
                print(f"  {field.title()}: {value}")
    else:
        print(f"No contacts found matching '{search_term}'")

# Main program loop
while True:
    display_menu()
    choice = input("Choose option (1-4): ")
    
    if choice == "1":
        add_contact()
    elif choice == "2":
        view_all_contacts()
    elif choice == "3":
        search_contacts()
    elif choice == "4":
        print("Goodbye!")
        break
    else:
        print("Invalid choice. Please enter 1-4.")

Example 2: Student Grade Tracker

# Student grade tracking system using dictionaries
students = []

def add_student():
    student = {
        "name": input("Student name: "),
        "id": input("Student ID: "),
        "math": int(input("Math grade: ")),
        "english": int(input("English grade: ")),
        "science": int(input("Science grade: "))
    }
    
    # Calculate average
    total = student["math"] + student["english"] + student["science"]
    student["average"] = total / 3
    
    # Determine grade letter
    if student["average"] >= 90:
        student["letter_grade"] = "A"
    elif student["average"] >= 80:
        student["letter_grade"] = "B"
    elif student["average"] >= 70:
        student["letter_grade"] = "C"
    elif student["average"] >= 60:
        student["letter_grade"] = "D"
    else:
        student["letter_grade"] = "F"
    
    students.append(student)
    print(f"Added {student['name']} (Average: {student['average']:.1f}, Grade: {student['letter_grade']})")

def display_report():
    if len(students) == 0:
        print("No students in database.")
        return
    
    print(f"\n=== Grade Report ({len(students)} students) ===")
    print(f"{'Name':<15} {'ID':<8} {'Math':<5} {'Eng':<5} {'Sci':<5} {'Avg':<6} {'Grade'}")
    print("-" * 55)
    
    for student in students:
        print(f"{student['name']:<15} "
              f"{student['id']:<8} "
              f"{student['math']:<5} "
              f"{student['english']:<5} "
              f"{student['science']:<5} "
              f"{student['average']:<6.1f} "
              f"{student['letter_grade']}")

# Demo usage
add_student()

display_report()

üö® Common Errors and How to Fix Them

Error 1: KeyError - Key doesn't exist

# Problematic code:
contact = {"name": "Alice", "phone": "123-456"}
print(contact["email"])  # ERROR: 'email' key doesn't exist

Fix: Always check if key exists first:

if "email" in contact:
    print(contact["email"])
else:
    print("No email available")

# Or use .get() method
email = contact.get("email", "No email")
print(email)

Error 2: Forgetting quotes around keys

# Problematic code:
contact = {name: "Alice"}  # ERROR: name should be "name"

Fix: Always put string keys in quotes:

contact = {"name": "Alice"}  # Correct

Error 3: Modifying dictionary during iteration

# Problematic code:
for key in contact:
    if key == "temp":
        del contact[key]  # Can cause issues

Fix: Create a list of keys to delete first:

keys_to_delete = []
for key in contact:
    if key.startswith("temp"):
        keys_to_delete.append(key)

for key in keys_to_delete:
    del contact[key]

üéØ Key Takeaways

  • Dictionaries store data as key-value pairs for organized, meaningful access
  • Keys must be strings (in quotes) and are used to access values
  • Values can be any data type (strings, numbers, lists, even other dictionaries)
  • Lists of dictionaries are perfect for storing multiple records (Merit level!)
  • .items() loops through key-value pairs
  • .keys() and .values() loop through keys or values only
  • Always check if keys exist before accessing them
  • Dictionaries are mutable - you can add, modify, and delete entries
