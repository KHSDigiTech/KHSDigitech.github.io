Sprint 1: Core Mechanics

Weeks 3-6 of Your Major Project

üéØ Sprint Goal

To create a playable prototype with the core game mechanics fully implemented.

üìã Sprint 1 Success Criteria

  • Player character with basic movement controls
  • Primary game mechanic implemented and functional
  • Simple test environment or level blockout
  • Basic win/lose conditions defined
  • Initial user feedback collected and documented

üìÖ Weekly Breakdown

Week 3: Setup & Player Movement
Focus: Initialize the game project in your chosen engine/framework. Implement the player character and basic movement.

Tasks to Complete:

  • Create the main project folder and structure
  • Implement player input for movement (e.g., keyboard controls)
  • Create a simple "player" object (can be a basic shape)
  • Write weekly journal entry
Week 4: Core Mechanic Implementation
Focus: Develop the main gameplay mechanic that makes your game unique.

Tasks to Complete:

  • Implement the primary mechanic (e.g., shooting, puzzle interaction, dialogue system)
  • Create a simple "test environment" or "blockout" level to test the mechanic
  • Write weekly journal entry
Week 5: Basic Game State
Focus: Define win/lose conditions to make the game playable.

Tasks to Complete:

  • Implement a simple objective (e.g., reach a goal area, destroy all targets)
  • Implement a failure condition (e.g., player health reaches zero, timer runs out)
  • Write weekly journal entry
Week 6: Testing & Reflection
Focus: Get initial feedback on the core feel of the game and reflect on the sprint.

Tasks to Complete:

  • Have at least one other person playtest the prototype
  • Log their feedback in the Feedback Log
  • Complete the Sprint 1 Reflection below
  • Write weekly journal entry

üí° Development Tips

Technical Advice

  • Start Simple: Use basic shapes (squares, circles) for your initial prototype
  • Focus on Feel: How responsive are the controls? Does movement feel good?
  • Version Control: Commit your changes to Git regularly (daily if possible)
  • Test Early: Play your own game constantly to catch issues early

Assessment Tips

  • Document Everything: Take screenshots of your progress weekly
  • Explain Decisions: In your journal, explain why you made certain technical choices
  • Show Iteration: Document how you improved mechanics based on testing
